  <name>Task 1: ImageService with resume, retry, versioning, and batch generation</name>
  <files>bookforge/images/service.py, tests/test_image_service.py, tests/conftest.py</files>
  <behavior>
    - test_generates_all_pages: Service calls provider.generate() for each page in book (mock provider). State updated per page with status="ok", image_path, image_version=1, generated_at.
    - test_skips_completed_pages: Pages with status="ok" in state are NOT passed to provider. Provider.generate() call count equals only the incomplete pages.
    - test_redo_specific_pages: With redo_pages=[3], only page 3 is regenerated even if status="ok". Other pages untouched.
    - test_version_increment: When page 1 has image_version=1 in state, redo generates page-01-v2.png (version 2). Original v1 file is NOT deleted.
    - test_retry_transient: Provider raises ReplicateError on first 2 attempts, succeeds on 3rd. Page completes successfully. Provider called 3 times.
    - test_retry_gives_up: Provider raises ReplicateError on all 3 attempts. Page state set to status="failed" with error message. Other pages still processed.
    - test_model_error_no_retry: Provider raises ModelError. Immediately fails (no retry). Provider called exactly once.
    - test_prompt_assembly: Request prompt = style_guide.build_prompt_prefix() + ". " + page.effective_prompt(). Reference images = resolved character reference_image paths from style guide.
    - test_batch_processing: With 7 pages, processes in batches (gather batches of 3-4). All pages eventually processed.
    - test_state_written_after_each_page: After each page generation, save_state is called (atomic write per page, not batch).
  </behavior>
  <action>
    Add to `tests/conftest.py`: fixtures for mock_provider (AsyncMock implementing ImageProvider), sample 3-page book, sample style_guide with 2 characters, temp book_dir with images/ subdir.

    Create `bookforge/images/service.py` with:
    - `BATCH_SIZE = 3` constant
    - `async def generate_all(book_dir: Path, book: Book, style_guide: StyleGuide, redo_pages: list[int] | None = None) -> dict`:
      - Load state via load_state(book_dir)
      - Initialize state["pages"] dict if missing
      - Get provider via get_provider(style_guide.image.provider, state.get("model_version"))
      - Resolve character reference image paths: `[book_dir.parent / "style-guides" / c.reference_image for c in style_guide.characters.values()]`
      - Build prompt prefix via style_guide.build_prompt_prefix()
      - Select pages to run: skip status="ok" unless in redo_pages
      - Process in batches of BATCH_SIZE using asyncio.gather(*tasks, return_exceptions=True)
      - For each page task: call `_generate_page()`
      - Return final state dict
    - `async def _generate_page(book_dir, page, prompt_prefix, ref_images, provider, state) -> None`:
      - Assemble prompt: f"{prompt_prefix}. {page.effective_prompt()}"
      - Determine next version number from state
      - Build output path: `book_dir / "images" / f"page-{page.number:02d}-v{version}.png"`
      - Create ImageRequest with prompt, ref_images, model_version from state
      - Call `_generate_with_retry(provider, request)` (3 attempts, exponential backoff)
      - Write image bytes to output path
      - Update state for this page: status="ok", image_path, image_version, seed, model_version, generated_at
      - Call save_state(book_dir, state) atomically
      - On failure: set status="failed", error message, save_state
    - `async def _generate_with_retry(provider, request, max_attempts=3) -> ImageResult`:
      - Retry loop: catch ReplicateError and httpx.TimeoutException, re-raise ModelError immediately
      - Exponential backoff: asyncio.sleep(2 ** attempt) between retries
      - Raise on final failure

    Write tests FIRST (RED), then implement (GREEN). Mock the provider entirely — no real API calls.
  </action>
