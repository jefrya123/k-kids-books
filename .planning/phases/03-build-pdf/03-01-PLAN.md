Purpose: This is the first half of the PDF pipeline — producing the HTML that WeasyPrint will consume. The renderer must handle three language editions from a single template and two format variants (screen/print CSS).

Output: `bookforge/build/renderer.py` with `render_book_html()`, Jinja2 templates, CSS files, bundled NotoSansKR font, sRGB ICC profile, and unit tests.

    1. Download NotoSansKR-Regular.otf (KR-only, NOT pan-CJK NotoSansCJK) from Google Fonts / Noto repository. Place in `bookforge/assets/fonts/`. This is ~4MB OFL-licensed.

    2. Download sRGB_v4_ICC_preference.icc from ICC.org (https://www.color.org/srgbprofiles.xalter). Place in `bookforge/assets/icc/`. This is ~4KB.

    3. Create `bookforge/assets/templates/css/base.css`:
       - @font-face for Noto Sans KR using `url()` with a PLACEHOLDER path (renderer will inject the real absolute path at render time via Jinja2 variable `{{ font_path }}`)
       - Base typography: .text-en uses Georgia serif, .text-ko uses 'Noto Sans KR' sans-serif
       - Both languages: same font-size (1.4rem), same weight (400), same line-height (1.5), same color (#2c2c2c) — per research Pattern 7
       - .page class: position relative, overflow hidden
       - .illustration class: position absolute, object-fit cover
       - .text-safe-zone: position absolute, bottom-anchored with semi-transparent background for legibility

    4. Create `bookforge/assets/templates/css/screen.css`:
       - @page { size: {{ trim_w }}in {{ trim_h }}in; margin: 0; }
       - .page dimensions match trim size exactly (no bleed)
       - .illustration sized to fill page exactly

    5. Create `bookforge/assets/templates/css/print.css`:
       - @page { size: {{ page_w }}in {{ page_h }}in; margin: 0; bleed: 0.125in; marks: none; }
       - Page dimensions = trim + 2*bleed per axis
       - .page dimensions match full page with bleed
       - .illustration extends 0.125in beyond page on all sides via negative positioning (per research Pattern 4)
       - .text-safe-zone offset accounts for bleed (safe_margin + bleed)

    6. Create `bookforge/assets/templates/book_page.html.j2`:
       - Loop over pages with page-break-after: always
       - Full-bleed illustration as <img> element (NOT CSS background — WeasyPrint issue #934)
       - Edition-aware text: `{% if edition != 'ko' %}` shows English, `{% if edition != 'en' %}` shows Korean
       - Image path constructed from page number: `{{ image_dir }}/page-{{ '%02d' % page.number }}.png`

    7. Create `bookforge/assets/templates/book_base.html.j2`:
       - Full HTML document structure with DOCTYPE, head, body
       - Includes base.css inline (via Jinja2 include or variable)
       - Includes format-specific CSS (screen.css or print.css) inline
       - Renders book_page.html.j2 content in body
       - Uses `{{ base_url }}` for resolving relative asset paths

    Note: CSS files should use Jinja2 variables for dimensions (injected by renderer) so they are NOT hardcoded to 8.5x8.5 — satisfying BLD-08 configurability requirement. The renderer will calculate all dimensions from BookMeta.trim_size and StyleGuide.layout.
  </action>

       def render_book_html(
           book: Book,
           edition: str,       # "en" | "ko" | "bilingual"
           style_guide: StyleGuide,
           fmt: str,           # "screen" | "print"
           book_dir: Path,     # for resolving image paths

       Implementation:
       - Parse `book.meta.trim_size` (e.g. "8.5x8.5") into width/height floats

       - Return rendered HTML string
